## Overview

When a subscriber clicks "Keep my subscription" in the retention chat, ChurnQ records the outcome, applies the offer in Stripe, and fires a `save.created` webhook event to all your enabled endpoints.

---

## What gets recorded

Every save creates a `SaveSession` record with the following data:

| Field | Description |
|-------|-------------|
| `sessionId` | Unique session ID |
| `tenantId` | Your ChurnQ tenant |
| `subscriberId` | Stripe customer ID passed in `ChurnQ.identify()` |
| `stripeSubscriptionId` | Stripe subscription the offer was applied to |
| `subscriptionMrr` | Monthly value at time of save |
| `offerType` | `discount`, `pause`, `extension`, `downgrade`, or `empathy` |
| `offerAccepted` | `true` |
| `discountPct` | Discount percentage (discount offers only) |
| `targetPriceId` | Stripe Price ID (downgrade offers only) |
| `ChurnQFee` | 15% of MRR saved |
| `feeBilledAt` | Timestamp when the fee was charged (set after `invoice.paid` for deferred offers) |
| `createdAt` | When the session was created |

Sessions are visible in **Dashboard → Recent Sessions**.

---

## The `save.created` webhook event

Fired immediately when the subscriber accepts an offer. If you have webhooks configured, all enabled endpoints receive this event.

### Payload

```json
{
  "event": "save.created",
  "timestamp": "2026-04-06T12:00:00.000Z",
  "data": {
    "tenant_id": "uuid",
    "subscriber_id": "cus_abc123",
    "subscriber_email": "user@example.com",
    "offer_type": "discount",
    "discount_pct": 25,
    "mrr_saved": 36.75
  }
}
```

### `data` fields

| Field | Type | Description |
|-------|------|-------------|
| `tenant_id` | `string` | Your ChurnQ tenant ID |
| `subscriber_id` | `string` | Stripe customer ID |
| `subscriber_email` | `string` \| `null` | Subscriber email if provided during identify |
| `offer_type` | `string` | `discount`, `pause`, `extension`, `downgrade`, or `empathy` |
| `discount_pct` | `number` \| `null` | Discount percentage  only present when `offer_type` is `discount` |
| `mrr_saved` | `number` | MRR retained (full MRR for pause/empathy, discounted amount for discount) |

---

## What happens in Stripe after a save

| Offer type | Stripe action |
|------------|--------------|
| `discount` | Repeating coupon applied to subscription for the configured number of months |
| `pause` | `pause_collection: { behavior: "mark_uncollectible" }` set on subscription |
| `extension` | Negative customer balance credit (2 weeks of MRR) |
| `downgrade` | Subscription price swapped to the target plan at next billing cycle |
| `empathy` | No Stripe action  conversation alone retained the subscriber |

---

## Fee billing

ChurnQ charges **15%** of the MRR saved per successful save. **Money movement** is **once per month** (1st of the month, UTC): one Stripe payment per workspace that bundles all eligible saves. Your dashboard **Billing** page shows **one row per charge** (Stripe payment ref).

**When the fee amount is finalized** (before that monthly charge):

| Offer type | When `fee` / outcome is finalized |
|------------|-----------------------------------|
| `empathy` | When the save is recorded (conversation retained the subscriber) |
| `pause` | Pre-calculated at save time; **outcome** confirmed on `invoice.paid` after pause ends |
| `discount` | Pre-calculated at save time; **outcome** and exact fee updated on `invoice.paid` at the discounted rate |
| `extension` | After next `invoice.paid` |
| `downgrade` | After first `invoice.paid` at the new plan price |

If the subscriber never pays (e.g. accepts a discount then cancels before a confirming invoice), **no fee is collected**  the save is not confirmed for billing.

## Receiving save events

To get notified when a subscriber is saved:

- **Slack / Discord**  connect in Dashboard → Connections. Alerts fire in real-time.
- **Webhooks**  add any HTTPS endpoint in Dashboard → Connections → Webhooks. See [Webhooks](/integrations/webhooks).
- **Zapier / Make**  paste your Catch Hook URL into the Zapier or Make card. See [Zapier](/integrations/zapier) or [Make](/integrations/make).
